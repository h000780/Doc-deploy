进阶主题

本章节记录了落地实施中需要关注的进阶特性，属于“Future-Proofing”范畴。

4.1 与 4.2 关键实现

4.1 字段依赖与计算完整性

Field Dependencies

场景: 公式字段（Formula）引用其他字段（如 fld_price * fld_qty）。

策略: 参考 Specialized View 的健壮性设计。

  • 非强一致性: 不强制禁止删除被引用字段。
  • 运行时容错: 当计算引擎发现依赖字段缺失时，结果返回 Error 或 Null，并在 UI 提示用户修复公式。
  • 理由: No-Code 平台应优先保证灵活性，严格外键约束会明显降低用户体验。

4.2 深度运行时校验

Advanced Runtime Validation

在 Chapter 2.2 的基础校验之上，建议对拦截器层继续增强：

  • Zod Schema Generation: 动态将 App Schema 转换为 Zod Object。
  • Validation Rules:
    • Number: 确保 typeof value === 'number'。
    • Select: 确保 value 在 options 列表中。
    • Date: 确保是 ISO8601 字符串。
  • Sanitization: 对非法输入优先“清洗”而非直接报错（如尝试 parseInt），提升 API 兼容性。
